# pyIRCX Changes - 2026-01-12

## Summary
Major refactoring to remove DEFAULT config block, fix mode inconsistencies, and transition from `reg_chans` to `registered_channels` table with JSON properties for persistent channel state.

---

## Configuration Changes

### 1. Removed DEFAULT Configuration Block
- **File:** `pyircx.py`
- **Lines:** 112-229 (deleted)
- **Change:** Removed hardcoded DEFAULT configuration dictionary
- **Reason:** Configuration should be provided by installation script, not hardcoded
- **Impact:** Server now requires config file to start (raises FileNotFoundError if missing)

### 2. Updated ServerConfig
- `__init__`: Changed `self.data = self._deep_copy(self.DEFAULT)` → `self.data = {}`
- `load()`: Now raises FileNotFoundError instead of creating default config

## Database Schema Changes

### 10. Removed reg_chans Table (Legacy)
- **Change:** Removed `reg_chans` table entirely
- **Reason:** Replaced with `registered_channels` + properties JSON

### 11. Added properties Column to registered_channels
- **File:** `pyircx.py:2377`
- **Schema:** Added `properties TEXT` column to store channel state as JSON
- **Migration:** Auto-adds column if doesn't exist (ALTER TABLE)

### 12. Channel Properties Now Persist for Registered Channels
- **What persists:** owners, hosts, voices, ACCESS lists, topic, keys, user_limit, onjoin/onpart, ban_list, modes
- **Stored as:** JSON in `registered_channels.properties` column
- **Methods:**
  - `Channel.get_properties_json()` - Serialize properties
  - `Channel.load_properties_json(json)` - Restore properties

---

## Architecture Changes

### 13. Channels Are Now Dynamic by Default
- **Removed:** `load_channels()`, `save_channels()`, `periodic_save()`
- **Behavior:**
  - Dynamic channels: Created on-demand, lost when empty
  - Registered channels: Properties persist in database, restored on JOIN

### 14. Updated Channel Registration
- **Files:** `pyircx.py:5618-5624` (REGISTER command), `pyircx.py:6882-6888` (Registrar service)
- **Change:** Save channel properties to `properties` JSON column when registering
- **Restores:** All channel state (owners, hosts, voices, ACCESS, topic, keys, etc.)

### 15. Updated Channel Loading
- **File:** `pyircx.py:2581-2590`
- **Change:** `load_registered_channel()` now loads properties from JSON
- **Behavior:** Registered channels restore full state from database

## Configuration Template Changes

### 16. Removed Modes from Config Template
- **File:** `pyircx_config.json`
- **Change:** Removed entire `modes` section from configuration template
- **Reason:** User/channel mode strings are server implementation details, not user configuration
- **Impact:** Modes are now hardcoded in pyircx.py (lines 545, 1420, 2627, 2631)
- **Deleted:** `modes.user`, `modes.channel`, `modes.channel_defaults`
- **Commit:** 562ac0b "Remove static modes section from config template"

### 17. Fixed Channel Unlock Button (MODE +z/-z)
- **File:** `pyircx.py:7820`
- **Change:** Allow services (+s) to set/unset MODE +z in addition to high staff
- **Before:** `if not user.is_high_staff():`
- **After:** `if not (user.is_high_staff() or user.is_service()):`
- **Reason:** System user (service) executes admin commands but didn't have permission to set MODE -z
- **Impact:** Web admin unlock button now works correctly
- **System user:** Remains as service (+s) only, does NOT have +a (admin) mode
- **Commit:** 290b746 "Fix channel unlock button - allow services to set MODE +z/-z"
